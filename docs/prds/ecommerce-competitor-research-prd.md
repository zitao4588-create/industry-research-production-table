# 电商竞品研究 Agent PRD

## 1. 基础信息

- Agent 名称：电商竞品研究 Agent
- 产品线：Industry Research Product Line / 行业研究生产台
- 模板 ID：ecommerce_competitor_research
- 目标客户：想进入或优化电商品类的个人卖家、小团队和服务商
- 使用人：创始人、运营、选品负责人、内容负责人
- 所属行业：电商 / 竞品研究 / 行业情报
- 当前阶段：MVP
- 负责人：industry-research-production-table

## 2. 一句话定位

这个 Agent 帮助电商团队在陌生行业研究场景下，先自动发现公开信息源、生成采集计划、mock 采集资料并建立多方向行业数据库，再生成竞品研究报告、机会地图和行业情报周报，减少人工搜索、收藏链接、复制资料和反复整理表格的动作。

## 3. 使用场景

- 客户现在怎么做：手动搜索品牌、打开网站、截图、复制商品信息、看评论和社媒内容。
- 当前最费时间的环节：信息分散在官网、Shopify 页面、Amazon、TikTok、YouTube、Blog、评论和表格里，最后只得到一堆链接。
- AI 介入后希望变成什么样：先建立行业数据库，再拆竞品赚钱方式，最后形成可复用的机会地图和 Markdown 报告。
- 不适合这个 Agent 的场景：需要实时爬虫、大规模代理池、自动投放广告、自动下单、支付或多租户 SaaS 的场景。

## 4. 研究方法

本模板融入文章《如何用 Codex 在 1 小时内进入一个陌生行业（完整版）》的核心方法：

1. 先解决数据采集，不让用户手动录入陌生行业资料。
2. 先建行业数据库，不只问一次性答案。
3. 先拆同行赚钱方式，再看供应链、广告素材和内容打法。
4. 同时研究信息源、竞品、网站结构、产品、关键词、用户痛点、内容生态、机会和周报。
5. 把零散资料放进知识地图，而不是只放收藏夹。
6. 从一次性搜索升级为持续情报系统，后续用 RSS、sitemap、竞品监控和内容监控自动更新。

## 7. 工作流拆解

1. 创建研究项目。
2. 自动发现信息源：公开搜索、竞品官网、Shopify、sitemap、RSS、用户导出 CSV。
3. 生成采集计划：采集目标、目标数据库、刷新频率和合规边界。
4. mock 执行采集：生成 crawl job、crawl run、raw document 和 extraction job。
5. 建立行业数据库：信息源、竞品、网站结构、产品、关键词、用户痛点、内容、机会、周报。
6. URL、CSV、手动文本作为补充证据进入数据库。
7. 抽取竞品信息：导航结构、产品分类、collection、tag、SEO、blog、社媒。
8. 抽取产品信号：价格、品类、卖点、标签、爆品线索。
9. 抽取用户痛点：高频抱怨、高频需求、高频问题、用户目标。
10. 抽取内容信号：爆款话题、渠道、内容类型、流量和转化线索。
11. 生成机会评分。
12. 人工审核结构化结果。
13. 生成 Markdown 报告。

## 8. 自动化与爬虫预留

第一版不直接抓取真实网页，只用 mock crawler 和 mock LLM 输出跑通流程。后续可通过 n8n 或爬虫工具自动化：

- 公开搜索：发现竞品官网、行业媒体、榜单和内容源。
- URL 抓取：官网、Shopify collection、product、blog、landing page。
- sitemap / RSS：优先作为低风险持续监控入口。
- CSV 导入：商品表、评论表、关键词表、社媒内容表。
- RSS 监控：Shopify Blog、Amazon News、TikTok Shop、Meta Ads、行业媒体。
- 竞品周报：每周统计新增产品、新增 collection、新增 landing page、新增 blog 和新增关键词。
- 内容监控：统计点赞、评论、转发、播放增长最快的内容。

## 10. 验收标准

- 能创建研究项目并选择 `ecommerce_competitor_research` 模板。
- 能输入行业、品类和市场。
- 能看到自动采集计划、mock 采集结果和九类数据库建设结果。
- 能把 URL、CSV 和手动文本作为补充资料导入；URL 留空时也能先通过公开搜索发现候选竞品官网。
- 能用 mock 结果跑完全部工作流步骤。
- 能查看竞品、产品信号、用户痛点、内容信号和机会评分。
- 能进行人工审核并生成 Markdown 报告。
- 配置 DeepSeek 后，能通过服务端 route 调用本机 DeepSeek 生成 Markdown 报告。
- 不接真实 OpenAI、不接复杂爬虫、不接登录、不接支付、不做复杂多租户；DeepSeek 仅用于公开资料抽取和报告生成节点。

## 11. 暂不做

- 登录：暂不做。
- 支付：暂不做。
- Docker：暂不做。
- SaaS 对外开放：暂不做。
- 复杂多租户：暂不做。
- 真实爬虫和代理池：暂不做。
- 真实 OpenAI 调用：第一版暂不做。
- DeepSeek 抽取结果自动入库后直接跳过人工审核：第一版暂不做。

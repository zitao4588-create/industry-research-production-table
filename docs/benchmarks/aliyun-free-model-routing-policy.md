# 阿里云免费 GLM / Kimi 模型路由策略

## 目标

尽可能利用已确认的免费额度，但不让低证据质量模型把内容写进正式报告。所有调用继续受“免费额度用完即停”、请求数、墙钟时间和 acceptedForReport 规则约束。

## 三层路由

### 每份报告固定使用

1. `Moonshot-Kimi-K2-Instruct`：快速压缩来源，只生成带 URL 和原文片段的内部摘要。
2. `glm-4.7`：复核证据绑定，只能否决或收窄结论。
3. `kimi-k2.6`：基于通过校验的数据库生成最终报告。

只有 `glm-4.7` 和 `kimi-k2.6` 可以参与 confirmed findings。任何模型输出都必须再经过确定性 validator。

### 抽样轮换使用

不是每份报告同时调用，而是按 runId 轮换 1-2 个，持续消耗免费额度并积累 benchmark：

- `glm-5.2`：高风险数字、排名和因果审计。
- `glm-5.1`：JSON schema 与章节格式检查。
- `glm-5`：标签和分类。
- `glm-4.6`：来源相关性二次筛选。
- `glm-4.5`：流式报告完整性检查。
- `glm-4.5-air`：流式正文噪音扫描。
- `kimi-k2.5`：仅生成待验证假设池。
- `kimi-k2-thinking`：反例、冲突和 kill rule 审计，不要求 JSON。

### 开发期使用

- `kimi-k2.7-code`：仅用于代码、测试和 validator 维护，不生成客户报告。

## 成本与停止规则

- 单份报告固定模型最多 3 个；轮换辅助模型最多 2 个。
- 每个辅助模型每份报告最多 1 次调用。
- 免费额度用完即停未开启、控制台显示无免费额度或 API 返回付费状态时，立即停止该模型。
- 连续 3 次 benchmark 证据完整度低于 50% 时，模型只保留格式/噪音等非事实任务。
- 返回空实体、只有 reasoning_content、报告截断或引用不可逐字匹配时，不进入正式结论。
- 不因为免费额度存在而重复生成同一份报告；额度用于分工和持续评测，而不是堆模型投票。

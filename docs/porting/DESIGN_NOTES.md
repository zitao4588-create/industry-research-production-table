## B. 视觉 / 微交互打磨清单(UX 文档未重点覆盖的)

### B-1　真实运行的「不确定态」也要好看
P0-A 最小版会引入 indeterminate 进度。别用裸 spinner。设计建议:
- 知识图谱在真实运行时保持「织网」动效,但进度未知时用**呼吸式微光 + 慢速粒子流**代替百分比环,百分比位置改显「采集中 · 已 Xs」。
- 步骤流用「当前步高亮脉冲 + 其余 pending」,不要假装精确百分比。诚实且不失质感。

### B-2　骨架屏要用「品牌化 shimmer」而非灰条
P0-B 的 `<Skeleton/>`:shimmer 的高光色用 `--accent-glow` 的极低透明度扫光(呼应 `btnShine`),圆角/间距复用 `.db-card`/`.stat-cell`,让加载态也是设计语言的一部分。

### B-3　数字进场只在「首次 done」播一次
`useCountUp` 现在每次进 Results 都会重跑。建议:同一 run 内只在 setup→done 的那一次播 count-up,切 tab/返回时直接显示终值(用一个 `hasAnimatedRef` 或把 run 完成时间戳作 key),避免重复动画显廉价。

### B-4　证据溯源弹层(P1-E)的设计规格
这是核心卖点,值得做精:
- 触发点视觉提示:可溯源的单元格 hover 时,数字旁浮现一个极小的「来源」角标(链接图标),告诉用户「可点」。
- 弹层结构:来源 URL(带 favicon/域名)+ 原文片段(引号块,高亮命中词)+ 置信度(用现有 `.score-bar` 语言)+「打开原文 ↗」。
- 用 `--glass` + `backdrop-filter`,与命令面板同一套弹层语言,保持一致。

### B-5　空态分语义(呼应 P2-J)
现在只有 `NeedRun` 一种空态且语义错位。建议三种,共用 `.empty` 骨架但文案/图标/CTA 不同:
- **未运行**(还没建库):`NeedRun`,CTA「去研究台」只切视图。
- **某库 0 条**(跑了但没采到):说明可能原因(源被限流/品类太窄)+「补充资料」「重试该步」。
- **检索无结果**(命令面板):已有,保持。

### B-6　运行模式术语统一(P2-K)
UI `Public + DeepSeek` / 设计稿 `Public+9router` / API `glm·9router·deepseek` legacy 三处不一致。**以 UX 文档 §3 的映射表为唯一对外口径**,UI 文案、tooltip、设置页说明统一过来,避免用户困惑。

### B-7　结果可操作的「轻动作」补全
除 P1-D 的下载/审核回写外,建议给机会表加行内轻动作(hover 出现):「加入监控周报」「导出该行 CSV」。让结果不止「可看」,而「可用」——呼应「生产台」这个名字。

### B-8　移动端(P2-H)别只是「能用」
≤720px 抽屉式侧栏之外:英雄区图谱在窄屏改为**静态成品图 + 降帧**(canvas 满帧在手机上费电),九库卡改单列、微图保留,统计条横向可滑。保证「精致」在手机上不打折。

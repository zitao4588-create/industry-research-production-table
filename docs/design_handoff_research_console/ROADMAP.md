# 待落地清单 · ROADMAP

> 给 Codex:这是设计阶段提出但**尚未实现**的事项,已按优先级排好。每条含「为什么 / 怎么做 / 落点」。当前原型完成的是**视觉与交互骨架**;下面是把它变成可对接真实 agent 的产品所需的工作。
>
> 优先级原则:**先做能在 agent 架构/IO 未定型时就落地、且能降低后续返工的事**(P0),再做依赖真实数据但价值高的(P1),最后是锦上添花(P2)。
>
> **⚠️ 2026-06-22 更新**:界面已移植进仓库,实际「接线」状态与本文件设想有出入(运行模式当前是装饰、真实后端已就绪但前端未接、缺失败态)。落地以 **`UX_OPTIMIZATION_HANDOFF.md`(同目录)** 这份 as-built 交接文档为准,本文件作为设计期背景参考。

## P0 · 现在就能做、且为接后端铺路

### P0-1　数据契约 + 适配层 ★最先做
- **为什么**:让 UI 与还没定型的 agent I/O 解耦,前后端并行。
- **怎么做**:见 `DATA_CONTRACT.md` §3。把 `data.js` 迁为 `fixtures.ts`;新建 `adapters/research.ts` 的 `adaptRun(raw)`;UI 只认 UI 模型。
- **落点**:新文件 `adapters/`、`fixtures/`;界面零改动。

### P0-2　running 态改为事件流驱动
- **为什么**:真实 agent 逐步吐事件,而非定时器。事件流版同时支持 mock 与真实。
- **怎么做**:见 `DATA_CONTRACT.md` §4 的 `RunEvent` 枚举。维护 `events[]`,从中派生步骤/百分比/计数器/信息源流。替换 `app.jsx` 中 `setInterval`+`TOTAL` 的模拟逻辑。
- **落点**:`Running` 组件改为消费 `events`;mock 提供假事件序列。

### P0-3　非 happy-path 状态(集成测试必撞)
- **为什么**:现在只有成功路径;真实会遇到 源发现为空 / 官网 403 / 抽取坏 JSON / LLM 超时 / mock 与真实不一致。
- **怎么做**:为每个区块补四态——
  - **加载骨架屏**(stat 条、数据库卡、表格的 shimmer 占位)
  - **空状态**(某库 0 条:给原因 + 重试/补充资料入口,复用现有 `.empty` 样式)
  - **错误态**(步骤失败:`run.error` 事件 → 红色步骤 + 错误详情 + 重试按钮)
  - **部分成功**(九个库**逐个填充**而非一次性出现,半截 run 也可用)
- **落点**:各结果组件加状态分支;新增 `<Skeleton/>`、`<ErrorState/>` 组件。

## P1 · 依赖真实数据、但价值高

### P1-1　开发者「运行检查器」抽屉(直接服务你现在的 I/O 测试)
- **为什么**:集成测试期间最顺手的工具。
- **怎么做**:右侧可收起抽屉,展示:原始 `RunEvent` 时间线、每步耗时、prompt/响应、token 数、**Mock ↔ 真实结果对比**。仅开发态显示(env flag)。
- **落点**:新组件 `<RunInspector/>`;读同一份 `events[]`。

### P1-2　证据溯源交互
- **为什么**:产品核心卖点是「可溯源」。
- **怎么做**:点任意机会/竞品/痛点单元格 → 弹层显示来源 URL、原文片段、置信度(数据形状见 `DATA_CONTRACT.md` §6)。UI 先做、字段先贯穿全链路(可先返回空数组)。
- **落点**:新增 `<EvidencePopover/>`;表格单元格挂 `evidenceId`。

### P1-3　置信度 / 数据质量标记体系
- **为什么**:让 agent 把质量信号透出,用户知道哪些可信。
- **怎么做**:定义视觉词汇并预留:`低证据 / 待复核 / 已过期 / 冲突`。复用现有 `status-pill` 风格,扩展 kind。
- **落点**:`components.jsx` 扩 `StatusPill`/新增 `QualityTag`。

### P1-4　微图全部接真实数据
- **为什么**:原型里数据库卡部分微图用示意数组(`micro.jsx` 的 `NodeMini`、`Blocks`、weekly `Spark`)。
- **怎么做**:`DbMicro` 已大多从 `IRP` 派生;把剩余几个也接上真实分布(源拓扑、结构层级、周报趋势)。
- **落点**:`micro.jsx` `DbMicro` 分支。

## P2 · 锦上添花 / 后续

### P2-1　结果可操作化
机会「转为产品/内容选题」「加入监控列表」「导出 CSV/PDF」等动作(目前报告仅「复制」)。

### P2-2　多项目 / 历史
研究项目列表接真实数据;支持打开历史 run、对比两次研究的差异(周报已是这个方向的雏形)。

### P2-3　周报订阅与推送
`weekly_intelligence_reports` 接定时任务;新增信号/监控列表可配置。

### P2-4　响应式深化与无障碍
现在桌面优先,有基础断点。需:窄屏侧栏抽屉化、表格横向滚动优化、命令面板移动端、完整键盘可达性与 ARIA、对比度核查。

### P2-5　主题/品牌收口
原型用 `tweaks-panel.jsx` 做预览调参——**生产请删除**,主色/明暗改用你仓库的设计系统;若需保留主色可配置,做成正式的偏好设置。

### P2-6　动效产品化
入场用 `html.booted` 门控、`prefers-reduced-motion` 已处理;迁移到框架时保留"可见态为基态、只从隐藏态入场"的原则,避免 SSR/首帧空白。

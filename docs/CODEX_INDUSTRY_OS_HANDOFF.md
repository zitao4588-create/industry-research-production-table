## 2. 本轮最重要的产品方向修正

### 已确认的新方向

- “护肤品”是合法的行业级输入，不是默认负例。
- 用户不需要把大行业继续缩小成某个 SKU 或窄品类。
- 系统应在内部拆解子市场、价格带、渠道、需求、品牌集群、产业链和监管问题。
- 市场/地区、时间范围和研究目标仍需要存在，但它们是研究坐标，不是要求用户缩小行业。
- “电商竞品研究”继续保留，但降为 Industry OS 中的一个研究模块，而不是整个产品的上位定义。

### 被新方向取代的旧假设

以下内容是历史证据或旧执行假设，不能再作为新产品方向的权威定义：

- Goal objective 文件把实例 12“护肤品”定义为模糊输入负例，并要求系统提示缩小品类。
- `scripts/run-industry-research-evidence-benchmark-v2.ts` 当前未提交 diff 中仍使用 `skincare-broad-negative`。
- `outputs/industry-research-benchmarks/evidence-benchmark-v3-post-kill-exploratory/post-kill-scorecard.md` 把“未识别输入过宽”列为护肤品失败原因。
- 旧 benchmark 的“至少 3 份文档、2 个品牌域名、1 个深页”仍可用于产品级竞品模块，但不能单独作为大行业报告的完成标准。

上述运行结果仍然有价值：它真实证明了当前链路会发生来源不足、品类串线和覆盖面失控；但不能由此推出“护肤品必须缩小”。正确结论是当前系统缺少行业规划与覆盖控制。

## 3. 原始方法论的正确落点

项目 PRD 引用的原始方法来自《如何用 Codex 在 1 小时内进入一个陌生行业（完整版）》：<https://aronhouyu.com/archives/480>。

应保留的核心不是“一次抓几个竞品网站”，而是：

1. 先解决陌生行业的数据和信息源问题。
2. 先建立行业数据库，再生成一次性答案。
3. 研究代表性竞争者如何赚钱，而不只是摘录产品页面。
4. 同时覆盖品牌、产品、关键词、社区、内容、商业模式、供应链、监管、趋势和机会。
5. 把证据组织成知识地图。
6. 从一次搜索升级为可持续更新的行业情报系统。

因此目标架构应是 `Industry OS -> 研究模块 -> 采集与证据 -> 综合报告 -> 持续更新`。现有电商竞品链路位于“研究模块/采集与证据”层。

## 5. 目标研究链条

### 阶段 1：研究坐标

输入大行业、市场/地区、时间范围、研究目标和可选补充资料。大行业本身保持完整。

### 阶段 2：Industry Planner

自动生成：

- 行业定义与边界；
- 子市场分类；
- 产业链；
- 价格带；
- 渠道；
- 消费者需求；
- 代表性商业模式；
- 监管和高风险问题；
- 研究问题；
- 模块计划；
- 来源角色策略；
- 覆盖矩阵；
- 代表性抽样计划；
- 预算、风险和停止条件。

### 阶段 3：行业广度扫描

按覆盖矩阵发现监管机构、行业协会、统计数据、可信研究机构、零售平台、代表品牌、社区、内容平台、产业链企业和关键词来源。

### 阶段 4：代表性抽样

按子市场、价格带、渠道和商业模式选择代表样本，避免随机搜索结果决定整个行业结构。

### 阶段 5：模块深挖

分别建立品牌、产品、用户需求、关键词、内容、商业模式、产业链、监管、趋势和机会数据库。电商竞品研究是其中一个模块。

### 阶段 6：跨模块综合

综合判断细分市场变化、价格带迁移、渠道变化、竞争集群、供需缺口、内容同质化和商业模式差异，并区分事实、信号、推断和假设。

### 阶段 7：行业报告与知识地图

输出可追溯的大行业报告、覆盖状态、证据缺口、机会假设和知识地图，而不是只拼接竞品页面摘要。

### 阶段 8：持续更新

用 sitemap、RSS、新品、内容、关键词、监管变化和跨 run diff 形成增量情报与周报。

## 6. 来源角色策略

来源不能只做全局“可信/不可信”判断，还要限制它能支持什么类型的结论：

| 结论类型 | 优先来源 | 不应单独支持的结论 |
|---|---|---|
| 市场规模与增速 | 政府统计、协会、可信研究机构、公司财报 | 单个品牌官网不能代表全行业规模 |
| 法规与标准 | 监管机构、正式法规、标准组织 | 博客和媒体摘要不能替代法规原文 |
| 品牌定位与产品 | 品牌官网、官方商城、可信零售渠道 | 品牌自述不能直接证明消费者需求 |
| 用户痛点与需求 | 评论、社区、用户调研、搜索信号 | 单条评论不能外推全市场需求规模 |
| 内容与流量趋势 | 内容平台、创作者数据、搜索趋势 | 页面标题不能直接证明转化效果 |
| 商业模式与供应链 | 财报、公司资料、产业媒体、供应链企业 | 营销文案不能证明盈利能力 |

G6 已把 `allowedClaimRoles` 等价契约接入 source/raw/evidence/review/report 全链路；它与 `acceptedForReport`、quote 唯一绑定、claim 完整性和人工审核同时生效，不能互相替代。

## 7. “护肤品行业报告”的目标结构

1. 研究范围、定义和证据边界；
2. 子市场与分类体系；
3. 产业链和商业模式；
4. 市场规模、增速和渠道结构；
5. 竞争格局与品牌集群；
6. 产品、功效和价格带；
7. 消费者人群、需求和痛点；
8. 搜索关键词、内容与流量生态；
9. 监管、技术、成分和趋势；
10. 可执行机会假设；
11. 反例、证据缺口和待验证问题；
12. 知识地图与持续监控计划。

## 9. 第一阶段验收结果

1. 输入 `industry=category=护肤品` 时，planner 接受它为行业级输入，不要求缩小品类。
2. 输出至少覆盖分类、产业链、价格带、渠道、需求、商业模式、监管、研究模块、来源角色、覆盖矩阵和抽样计划。
3. 电商竞品研究明确作为模块存在，而非上位产品定义。
4. 每个研究模块都说明研究问题、需要的来源角色、覆盖要求和完成状态。
5. 来源角色能阻止“品牌自述 -> 全行业需求/规模”的错误外推。
6. 护肤品 fixture 不得混入运动服饰品牌；若使用跨行业案例，必须明确标成 business-model analogy，不能算护肤品竞争者。
7. 规划器输出可序列化、可复现、可测试；失败时保留缺口，不用 mock 结论补满。
8. `pnpm check` 通过；如涉及 UI，再补 `pnpm build`，但第一阶段默认不改 UI。

上述 8 项均已通过，当前完成等级为 **C2：Industry Planner 本地可运行**。这不表示完整行业研究已经交付、生产已经升级或护肤品报告已经完成。
